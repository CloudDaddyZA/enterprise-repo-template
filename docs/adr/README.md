# Architecture Decision Records (ADRs)

## What are ADRs?

Architecture Decision Records capture important architectural decisions made along with their context and consequences. They provide a lightweight way to document the "why" behind technical decisions.

## Template

Use the [ADR template](./template.md) when creating new records.

## Statuses

- **Proposed**: Under discussion
- **Accepted**: Decision made and in effect
- **Deprecated**: No longer applies (superseded or reversed)
- **Superseded**: Replaced by a newer ADR
